Change History:
6.0 - Worked around buffer overflow with large cset and using -fmt option to lsactivity.

From customer feedback, the following two features
(thank you patrick.renaud@ericsson.com):

1) Renamed move action to redo.
2) Implemented a new operation called findmerge. This will do a real merge
of the cset (not a redo). Used "findmerge" instead of "merge" to
avoid older customers accidentally using a -m option and getting
different functionality.

5.4 – Fixed a minor defect with the –print option.

5.3 – Support for spaces in change set elements.

5.2 – Tested on Linux 2002. Support for Linux/Unix. Added prompting in using clearprompt.
Removed need to require Clearcase.pm. Move functionality from that module into cset.pl.
Now, only one file needed to deploy. Added a version number. Added support for CQ
enabled projects by getting the headline and prompting with that as well as the CQ ID.
Quotes and other grammar replaced with spaces in headline that gets prompted, as
clearprompt does not display these characters correctly.

All previous versions:

Added the –g option to support having the merge command use graphical merge. Otherwise, it will spawn a box with command line diff style merge. This is annoying when bolting into explorer. However, a side effect is that it will always bring up the GUI, even if the merge is automatic. When bolting into CC explorer or Windows explorer, you may want to add multiple items, one for GUI one for not. Also, when attaching to explorer, be careful where you attach. If you attach to a tree in the left pane, the script does not get its home directory set correctly. The only place I found where it works is on a checked in element on the ride side of the pane.

Added support for directories as well as files. Made work on Unix and Windows.
